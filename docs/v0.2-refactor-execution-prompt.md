# llmcaller-codex-go v0.2 Refactor Execution Prompt

Implement the caller refactor specified by
[v0.2-refactor-plan.md](v0.2-refactor-plan.md) in the current
`llmcaller-codex-go` repository.

Read the complete plan before editing. It is authoritative for ownership,
mirrored upstream contracts, target local API, projection, schema policy,
migration, dependency gates, and acceptance.

## Rules

- Recheck current git state, exports, `go.mod`, and resolved upstream APIs.
- A symbol in a mirrored target block is not assumed available merely because
  the plan names it. Confirm it with the resolved module source or `go doc`.
- Before implementation, byte-compare both mirrored blocks with their
  authoritative upstream plans. Any mismatch is a blocker requiring contract
  reconciliation, not an invitation to guess.
- Do not use local `replace` or committed `go.work` to conceal missing released
  APIs. Temporary all-heads workspaces are validation only.
- Keep adapter ownership narrow: schema policy, exact request/result
  translation, defaults, and named safety profile.
- Preserve exact SDK details and partial results; do not introduce copied Codex
  enums/results, raw metadata bags, generic retry, workflow, or transport code.
- Enforce equality between neutral `Execution.ProviderName` and non-nil typed
  details identity on every full or partial response.
- Preserve unknown schema keywords by JSON value semantics, not bytes. Run the
  complete Go type compatibility matrix as a release gate.
- Preserve unrelated user changes and untracked files.
- Every public change includes behavior tests, docs, migration, CHANGELOG,
  compatibility matrix, examples, and handwritten export allowlist.

## Execution Order

1. Verify both contract mirrors and actual dependency availability.
2. Implement dependency-independent schema policy work and tests.
3. If real upstream target tags resolve, update `go.mod` and implement exact
   calls, streams, details, evidence projection, partial results, and profile.
4. Add compile-time assertions, minimum/all-heads/tag canaries, compiled
   examples, docs, release metadata, and export enforcement.
5. Audit every requirement; green unit tests alone are insufficient.

## Validation

Run at minimum:

```sh
GOWORK=off go test ./...
GOWORK=off go vet ./...
```

Also run:

- exact contract-block byte comparisons;
- standalone tests against required versions;
- temporary all-heads integration;
- real upstream RC/stable-tag canary;
- compile-checked three-layer example and README snippets;
- handwritten export allowlist and dependency-direction tests.

Do not push, tag, publish, or modify upstream repositories without explicit user
authorization. If upstream target tags do not exist, stop only after completing
all dependency-independent work and report the exact missing module versions and
symbols.

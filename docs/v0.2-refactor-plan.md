# llmcaller-codex-go v0.2 Refactor Plan

Status: implemented, standalone/integration verified, and released as v0.2.0

Contract revision: `llmcaller-codex-v0.2-r3`

## Purpose

`llmcaller-codex-go` adapts provider-neutral structured-output calls from
`llmkit-go` to exact Codex lifecycle operations from `codexsdk-go`.

It owns Codex structured-output schema policy, Codex request/result projection,
construction-time Codex defaults, and named Codex safety profiles. It does not
own business validation, generic retry, workflow, Codex transport, or app-server
lifecycle.

## Verified v0.1 Baseline Facts

Before this refactor, the module required `llmkit-go v0.1.0` and
`codexsdk-go v0.1.0`. Its public API had `ThreadStarter`, field-by-field `Options`, `New`,
`Caller.Call`, `ReadOnlyEphemeralOptions`, and
`StrictOutputSchemaFromJSON`.

The v0.1 `Call` returned only `FinalResponse`; the named read-only helper did not
set a sandbox; schema normalization forced every property to required.

The target upstream symbols below did not exist in the v0.1 tags. They are now
resolved from real `llmkit-go v0.2.0` and `codexsdk-go v0.2.0` tags
without `replace` or a committed `go.work`. A temporary all-heads `go.work`
remains integration evidence only.

## Target Handwritten Public API

This section is normative for this repository.

```go
package codexcaller

var (
	ErrNilThreadRunner   error
	ErrMissingSchemaJSON error
)

type ThreadRunner interface {
	Start(context.Context, codexsdk.StartThreadRunRequest) (codexsdk.StartedThreadRun, error)
	StartStream(context.Context, codexsdk.StartThreadRunRequest) (*codexsdk.Stream[codexsdk.StartedThreadRun], error)
}

type Options struct {
	Runner   ThreadRunner
	Defaults codexsdk.StartThreadRunRequest
}

type Details struct {
	Run codexsdk.StartedThreadRun
}

func (Details) ProviderName() string

type Caller struct { /* opaque immutable configuration */
}

func New(Options) (*Caller, error)
func ReadOnlyEphemeralOptions(ThreadRunner) Options

func (c *Caller) Call(
	context.Context,
	llmadapter.Request,
) (llmadapter.Response, error)

func (c *Caller) CallDetailed(
	context.Context,
	llmadapter.Request,
) (codexsdk.StartedThreadRun, error)

func (c *Caller) CallStream(
	context.Context,
	llmadapter.Request,
) (*codexsdk.Stream[codexsdk.StartedThreadRun], error)

func StrictOutputSchemaFromJSON(json.RawMessage) (protocolv2.OutputSchema, error)

type SchemaPolicyError struct {
	Path string
	Kind string
	Err  error
}

func (e *SchemaPolicyError) Error() string
func (e *SchemaPolicyError) Unwrap() error
```

`ThreadRunner` is consumer-owned and intentionally contains only the two
methods the adapter uses. `New` rejects a nil runner and clones all mutable
defaults. `Defaults.Turn.ThreadID`, `Input`, and `OutputSchema` are
adapter-owned and must be zero; conflicts fail at construction rather than being
silently ignored. Every other exact generated field is available without a
copied adapter option model.

`CallDetailed` is the single exact execution core. It converts the neutral JSON
schema, builds exact generated `UserInput`, fills adapter-owned turn fields, and
calls the runner. `CallStream` uses the same construction path. `Call` projects
the exact run into `llmadapter.Response` and stores the same immutable run in
`Details`.

Projection rules:

- `FinalResponse` comes from `Run.FinalResponse`.
- `Execution.ProviderName` and `Details.ProviderName()` are both the literal
  stable identity `codex` and must be equal on every response with details.
- Effective model starts with `Start.Model` and follows exact model-reroute
  notifications in arrival order; no notification means the start model.
- Neutral usage maps from `Run.Usage.Total`; nil exact usage maps to nil neutral
  usage. Exact total/context-window facts remain in `Details`.
- Diagnostics, warnings, IDs, items, sandbox, approval, CWD, service tier,
  notifications, and exact configuration remain only in `Details`.

If the SDK returns a partial run with an error, `CallDetailed` returns both.
`Call` maps every available neutral fact and `Details` before returning the same
cause chain. No successful upstream fact is replaced with a zero result because
an error also exists.

## Schema Policy

`StrictOutputSchemaFromJSON` traverses every subschema position supported by the
current generated `protocolv2.OutputSchema` validator. Unknown keywords and
their JSON values are preserved semantically unless the typed parser rejects
them. Object key order, whitespace, number spelling, and escape spelling may
change during parse and serialization; byte-for-byte preservation is not part
of the contract and does not justify a token/span-preserving AST.

An optional property may be promoted to required only when its schema already
admits null after resolving local references. Otherwise return
`SchemaPolicyError` with a stable kind and JSON pointer. External references,
unresolvable references, and cycles fail closed before starting Codex.

The transformer must not use an incomplete typed struct that drops unknown
keywords, and must not silently change optional/nullable/required semantics.

### Go Type Compatibility Matrix

The integration canary and migration guide must classify at least these shapes:

| Go/schema shape | Required behavior |
| --- | --- |
| Non-`omitempty` scalar | Remains required and non-nullable |
| Pointer with `omitempty` | Accepted only when the actual generated schema admits null; otherwise `SchemaPolicyError` |
| Non-pointer scalar with `omitempty` | `SchemaPolicyError`; absence cannot be replaced by null without changing semantics |
| Nested optional property | Apply the same rule recursively at the exact JSON pointer |
| Optional property through local `$ref` | Resolve locally, then promote only if the resolved schema admits null |
| Recursive/cyclic local reference | Fail closed with stable `SchemaPolicyError` kind and path |
| Unknown keyword | Preserve its JSON value semantics through transformation |
| External or unresolvable `$ref` | Fail closed before starting Codex |

The canary must use `llmschema.SchemaJSONFor` from the exact llmkit version under
test rather than assuming Go pointer behavior. Migration documentation includes
concrete accepted and rejected Go struct examples and the resulting
`SchemaPolicyError` path/kind.

## Safety Profile

`ReadOnlyEphemeralOptions` is justified policy, not syntax sugar. It sets exact
generated values for:

- ephemeral thread creation;
- read-only thread sandbox mode;
- read-only turn sandbox policy;
- never-approve policy at every relevant thread/turn scope.

Tests assert the returned exact `ThreadStartResponse.Sandbox`, approval policy,
and nested `Thread.Ephemeral`, not only requested fields. If the effective
response is not read-only, never-approve, and ephemeral, the helper cannot
retain the read-only ephemeral name.

## Migration

- Change `New` to return `(*Caller, error)` in the pre-v1 v0.2 release.
- Remove the v0.1 field-by-field Model/CWD/Effort/approval options in this
  pre-v1 release; retaining them would contradict the normative API and create
  a second source for exact defaults.
- Document direct migration from every removed field to its generated
  `Options.Defaults` location.
- Keep `StrictOutputSchemaFromJSON`, but document its stricter behavior.

## Implementation Sequence

1. Implement semantic schema transformation and typed failure tests without
   waiting for upstream releases.
2. Prepare fake contract tests against local test doubles.
3. Wait for real llmkit and SDK RC/stable tags containing the mirrored contracts.
4. Update `go.mod` without `replace`; implement `Details`, exact calls, streams,
   evidence projection, and partial-result behavior.
5. Correct the safety profile and migrate options.
6. Add minimum/all-heads/RC canaries, compiled examples, docs, CHANGELOG, release
   metadata, and handwritten export allowlist.

# v0.4 Migration

v0.4.0 changes `Caller.CallStream` from returning
`*codexsdk.Stream[codexsdk.StartedThreadRun]` to returning the adapter-owned
`*codexcaller.Stream`. This is a documented pre-v1 breaking API change.

Use the adapter wrapper's `Next`, `Notification`, `Result`, `Wait`, `Err`, and
`Close` methods for normal observation. In particular, use `Wait` to receive
the complete exact result and joined terminal causes, or pair `Result` for the
exact snapshot with `Err` for the joined causes. `Wait` and `Err` enforce the
named effective-profile postcondition; `Result` preserves the exact snapshot
without validating it on its own.

When a lower-level API specifically requires the concrete SDK stream, use the
adjacent typed escape hatch:

```go
stream, err := caller.CallStream(ctx, request)
if err != nil {
	return err
}

sdkStream := stream.SDKStream()
```

Using `SDKStream` does not perform adapter-owned terminal profile validation.
Continue to observe terminal completion through the adapter wrapper whenever
the named profile contract matters.

# Migrating To v0.2

v0.2 replaces the projected v0.1 Codex option and result surface with exact
generated SDK request and result types. It is a pre-v1 breaking release.

## Construction

`New` now returns `(*Caller, error)` and accepts exact defaults:

```go
options := codexcaller.ReadOnlyEphemeralOptions(runner)
options.Defaults.Thread.Model = protocolv2.Value(model)
options.Defaults.Thread.CWD = protocolv2.Value(workspace)
options.Defaults.Turn.Effort = protocolv2.Value(protocolv2.ReasoningEffort("low"))

caller, err := codexcaller.New(options)
```

The v0.1 field-by-field `Options` fields and `ThreadStarter` interface are
removed. Use `Options.Defaults` and the consumer-owned `ThreadRunner` interface.
`Turn.ThreadID`, `Turn.Input`, and `Turn.OutputSchema` are adapter-owned and must
remain zero in defaults.

| v0.1 option | v0.2 exact default |
| --- | --- |
| `Model` | `Defaults.Thread.Model` |
| `CWD` | `Defaults.Thread.CWD` |
| `Effort` | `Defaults.Turn.Effort` |
| thread approval | `Defaults.Thread.ApprovalPolicy` |
| turn approval | `Defaults.Turn.ApprovalPolicy` |
| approvals reviewer | `Defaults.Thread.ApprovalsReviewer` or `Defaults.Turn.ApprovalsReviewer` |
| ephemeral | `Defaults.Thread.Ephemeral` |

## Results And Errors

Use `Call` for the neutral projection, `CallDetailed` for the exact SDK result,
and `CallStream` for exact notifications. `Call` exposes the same exact run via
`codexcaller.Details`.

An error no longer implies that no provider evidence exists. Inspect the
returned response or detailed run first, then handle the error. Error wrapping
preserves `errors.Is` and `errors.As` behavior.

## Structured Output Compatibility

The v0.1 transformer made every object property required, which silently
changed optional non-nullable semantics. v0.2 rejects that shape.

Accepted required scalar:

```go
type Output struct {
	Summary string `json:"summary"`
}
```

Accepted optional pointer when the exact `llmschema.SchemaJSONFor` output admits
null:

```go
type Output struct {
	Summary string  `json:"summary"`
	Note    *string `json:"note,omitempty"`
}
```

Rejected optional non-nullable scalar:

```go
type Output struct {
	Summary string `json:"summary"`
	Score   int    `json:"score,omitempty"`
}
```

The rejected example returns `*codexcaller.SchemaPolicyError` with kind
`optional_non_nullable` and path `/properties/score`. Nested paths use escaped
JSON Pointer tokens. Local references are resolved; external, unresolved,
dynamic, and cyclic references fail before the runner starts.

Do not infer compatibility from Go pointer shape alone. Release canaries use
`llmschema.SchemaJSONFor` from the exact resolved `llmkit-go` version.

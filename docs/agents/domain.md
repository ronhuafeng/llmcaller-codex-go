# Domain docs

This repository uses a single-context domain-doc layout.

Before exploring a domain area, read `CONTEXT.md` and relevant ADRs under
`docs/adr/` when they exist. Their absence is not an error; proceed silently.

Use glossary vocabulary consistently. If a proposed change conflicts with an
ADR, surface the conflict rather than silently overriding the recorded
decision. Create glossary entries or ADRs lazily through domain-modeling only
when a term or durable trade-off is actually resolved.
